# SpacetimeDB Database Schema

This document outlines the database schema, reducers, and data structures required to support the Courier Notifications project using SpacetimeDB.

## Core Concepts & SpacetimeDB Mechanics

SpacetimeDB uses strongly typed tables and "Reducers" (server-side functions) to interact with the data. 

### Data Types

*   **Identities**: Built-in cryptographic identities provided by SpacetimeDB.
*   **Strings**: Standard text fields.
*   **Enums**: Used for roles and status flags.
*   **Timestamps**: UNIX epoch timestamps for tracking creation, updates, and message send times.

---

## Tables / Models

### 1. `User`
Stores information about the people logging into the React App/PWA.
*   `identity: Identity` (Primary Key, Unique) - The SpacetimeDB identity of the user.
*   `email: Option<String>` - Used for email/PIN login.
*   `google_id: Option<String>` - Used for Google OAuth.
*   `passkey_credential_id: Option<String>` - Used for WebAuthn passkey login.
*   `name: String` - User's display name.
*   `push_token: Option<String>` - Firebase Cloud Messaging token for push notifications.
*   `created_at: u64`

### 2. `Venue`
A logical grouping of channels and users.
*   `venue_id: u32` (Primary Key, Auto-increment)
*   `name: String`
*   `owner_identity: Identity` (Foreign Key -> User.identity)
*   `created_at: u64`

### 3. `Channel`
A communication room inside a Venue.
*   `channel_id: u32` (Primary Key, Auto-increment)
*   `venue_id: u32` (Foreign Key -> Venue.venue_id, Indexed)
*   `name: String`
*   `minimum_role_to_view: ChannelRole` (Enum: Owner, Admin, Moderator, Member) - Who can see this channel.
*   `message_max_age_hours: u32` - How long members can see messages (default 4).
*   `created_at: u64`

### 4. `VenueMember`
Links a User to a Venue and stores their connection metadata.
*   `venue_id: u32` (Indexed)
*   `user_identity: Identity` (Indexed)
*   *(Note: The combination of `venue_id` and `user_identity` must be unique)*
*   `join_date: u64`
*   `last_seen: u64`
*   `is_blocked: bool` - If true, forcefully demotes user to 'Member' across all channels in this venue.

### 5. `ChannelMemberRole`
Defines a specific user's role in a specific channel. (Roles are per-channel, as per spec).
*   `channel_id: u32` (Indexed)
*   `user_identity: Identity` (Indexed)
*   *(Note: The combination of `channel_id` and `user_identity` must be unique)*
*   `role: ChannelRole` (Enum: Admin, Moderator, Member)

### 6. `NotificationFilter`
Stores user preferences for push notifications in a specific channel.
*   `channel_id: u32` (Indexed)
*   `user_identity: Identity` (Indexed)
*   `filter_type: NotificationFilterType` (Enum: EnableAll, DisableAll, FilteredText)
*   `filter_texts: Vec<String>` - List of strings to match against message content.

### 7. `MessageTemplate`
Templates for sending formatted messages.
*   `template_id: u32` (Primary Key, Auto-increment)
*   `channel_id: u32` (Foreign Key -> Channel.channel_id, Indexed)
*   `name: String`
*   `description: String`
*   `fields_json: String` - JSON array containing the structural definition of the fields (Regex, before/after text, optionality, ordering). Stored as a string to avoid complex nested struct limitations natively, decoded on the webapp.

### 8. `Message`
The core message entity.
*   `message_id: u32` (Primary Key, Auto-increment)
*   `channel_id: u32` (Foreign Key -> Channel.channel_id, Indexed)
*   `sender_identity: Identity` (Foreign Key -> User.identity)
*   `template_id: Option<u32>`
*   `content: String` - The collapsed/final string representation of the template fields.
*   `sent_at: u64`

### 9. `DisplayDevice`
Represents an Electron Desktop App connected to a Venue.
*   `display_id: u32` (Primary Key, Auto-increment)
*   `uid: String` (Unique) - The GUID generated by the Electron app.
*   `venue_id: u32` (Foreign Key -> Venue.venue_id, Indexed)
*   `name: String` - Friendly name (e.g., "Lobby TV")
*   `registered_at: u64`
*   `last_connected_at: u64`

### 10. `DisplayPairingPin`
Temporary pins for linking DisplayDevices to Venues.
*   `pin: String` (Primary Key, e.g., "123456")
*   `display_uid: String`
*   `expires_at: u64`

### 11. `MessageDeliveryStatus`
Tracks the delivery state of a message on a specific DisplayDevice.
*   `message_id: u32` (Indexed)
*   `display_id: u32` (Indexed)
*   *(Note: The combination of `message_id` and `display_id` must be unique)*
*   `status: DeliveryStatus` (Enum: Enqueued, InProgress, Shown)
*   `updated_at: u64`

## Core Reducers (API surface)

### User Authentication & Management
*   `login_or_create_user(email: Option<String>, google_id: Option<String>, name: String)`
*   `register_passkey(credential_id: String, public_key_bytes: Vec<u8>)`
*   `login_with_passkey(credential_id: String, webapp_data_json: String, authenticator_data: String, signature: String)`
*   `update_push_token(token: String)`

### Venue & Channel Management
*   `create_venue(name: String)`
*   `update_venue(venue_id: u32, new_name: String)`
*   `delete_venue(venue_id: u32, confirmation_name: String)`
*   `create_channel(venue_id: u32, name: String, min_role: ChannelRole, max_age_hours: u32)`
*   `delete_channel(channel_id: u32, confirmation_name: String)`
*   `update_channel_settings(channel_id: u32, new_name: String, min_role: ChannelRole)`

### Membership & Permissions
*   `join_venue(venue_id: u32)` - Called via invite link.
*   `leave_venue(venue_id: u32)`
*   `set_channel_role(channel_id: u32, target_identity: Identity, new_role: ChannelRole)`
*   `block_user(venue_id: u32, target_identity: Identity)`
*   `unblock_user(venue_id: u32, target_identity: Identity)`

### Notifications
*   `update_notification_filter(channel_id: u32, filter_type: NotificationFilterType, texts: Vec<String>)`

### Messaging
*   `create_message_template(channel_id: u32, name: String, description: String, fields_json: String)`
*   `update_message_template(template_id: u32, fields_json: String)`
*   `delete_message_template(template_id: u32)`
*   `send_message(channel_id: u32, content: String, template_id: Option<u32>)`
*   `delete_message(message_id: u32)`
*   `repeat_message(message_id: u32)` - Creates a new message copy.

### Electron Display Sync
*   `create_display_pin(display_uid: String)` - Called by the unauthenticated Electron App.
*   `register_display_to_venue(pin: String, venue_id: u32, name: String)` - Called by the authenticated PWA Owner/Admin.
*   `display_connect(display_uid: String)` - Called by the Electron App on boot to signal it is online and fetch missed messages.
*   `update_message_delivery_status(uid: String, message_id: u32, status: DeliveryStatus)` - Called by the Electron App to report progress.
